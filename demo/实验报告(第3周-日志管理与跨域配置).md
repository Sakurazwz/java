## 三、实验原理

1. **AOP 日志记录原理**  
   通过自定义 `@Log` 注解标记目标方法，使用 `@Aspect` + `@Around` 在方法执行前后统一采集日志信息，包括访问 IP、请求 URL、方法参数、执行耗时、执行结果与异常信息。

2. **全局跨域（CORS）原理**  
   浏览器在跨源请求时会先发起预检请求（OPTIONS）。在 Spring MVC 中可通过 `WebMvcConfigurer#addCorsMappings` 统一配置允许的源、方法、请求头、凭证与缓存时间，后端据此返回 `Access-Control-Allow-*` 响应头。

3. **AOP 通知类型原理**  
   - `@Before`：目标方法执行前触发
   - `@After`：目标方法执行后触发（无论是否异常）
   - `@AfterReturning`：目标方法正常返回后触发
   - `@AfterThrowing`：目标方法抛出异常后触发

## 四、实验内容与步骤

### 1. 实现日志切面

1. 创建 `@Log` 注解（`annotation/Log.java`），定义 `value` 与 `module` 字段用于描述操作。
2. 创建 `LogAspect`（`aspect/LogAspect.java`），使用 `@Pointcut("@annotation(com.example.demo.annotation.Log)")` 拦截带注解的方法。
3. 在环绕通知中记录：
   - 请求 IP（`IpUtils`）
   - 请求路径与 HTTP 方法
   - 调用方法签名与参数
   - 执行耗时
   - 成功结果或异常信息
4. 在 `UserController` 的 3 个方法上添加 `@Log` 注解：
   - `getUser(Long id)`
   - `getUserList()`
   - `register(UserRegisterDTO dto)`

### 2. 配置 CORS

1. 创建 `WebMvcConfig`（`config/WebMvcConfig.java`）并实现 `WebMvcConfigurer`。
2. 全局跨域配置：
   - 路径：`/**`
   - 允许来源：`http://localhost:5173`
   - 允许方法：`GET/POST/PUT/DELETE/OPTIONS`
   - 允许请求头：`*`
   - 允许凭证：`true`
   - 预检缓存：`3600` 秒

### 3. 测试各种通知

1. 创建 `DemoAspect`（`aspect/DemoAspect.java`），定义针对 `UserController` 的切入点。
2. 分别实现并验证：
   - `@Before`
   - `@After`
   - `@AfterReturning`
   - `@AfterThrowing`
3. 通过测试反射读取方法注解，确认通知类型配置完整。

### 4. 执行自动化测试

1. 执行全量测试（demo 模块）：

```bash

```

2. 执行第 3 周定向测试：

```bash

```
