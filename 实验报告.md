<h1>第3周日志管理与跨域配置实验报告</h1>

<h2>实验目的</h2>
<p>理解 AOP 在日志管理中的应用，掌握通过切面记录请求日志的方法；理解跨域问题产生原因，并使用 Spring Boot 完成全局 CORS 配置。</p>

<h2>实验内容</h2>
<ul>
    <li>实现 <code>@Log</code> 注解与 <code>LogAspect</code> 日志切面，记录请求 IP、参数、耗时等信息</li>
    <li>在 <code>UserController</code> 的查询单个用户、查询用户列表、用户注册三个方法上添加日志注解</li>
    <li>使用 <code>WebMvcConfigurer</code> 完成全局跨域配置，允许 <code>http://localhost:5173</code> 访问</li>
    <li>实现并验证 <code>@Before</code>、<code>@After</code>、<code>@AfterReturning</code>、<code>@AfterThrowing</code> 四种通知</li>
</ul>

<h2>实验步骤</h2>
<ol>
    <li>创建日志注解 <code>com.example.demo.annotation.Log</code></li>
    <li>创建日志切面 <code>com.example.demo.aspect.LogAspect</code>，通过环绕通知统一记录请求与响应日志</li>
    <li>在 <code>UserController</code> 指定接口上添加 <code>@Log</code> 注解</li>
    <li>创建 <code>com.example.demo.config.WebMvcConfig</code>，在 <code>addCorsMappings</code> 中配置全局跨域规则</li>
    <li>创建 <code>com.example.demo.aspect.DemoAspect</code>，实现多种通知并观察执行顺序</li>
    <li>编写并运行测试，验证日志注解、CORS 策略和通知实现是否符合要求</li>
</ol>

<h2>测试结果</h2>
<p>本次实验通过 Maven 测试验证了课后作业要求，核心功能工作正常。具体如下：</p>
<ul>
    <li>用例1：校验 <code>UserController</code> 三个目标方法具备 <code>@Log</code> 注解，验证通过。</li>
    <li>用例2：OPTIONS 预检请求来自 <code>http://localhost:5173</code> 时允许跨域访问，返回 200，验证通过。</li>
    <li>用例3：OPTIONS 预检请求来自其他来源（如 <code>http://localhost:3000</code>）时被拒绝，返回 403，验证通过。</li>
    <li>用例4：校验演示切面中四种通知（前置、后置、返回、异常）均已实现，验证通过。</li>
</ul>

<h2>实验结论</h2>
<p>已完成第3周课后作业要求。通过 AOP 可将日志等横切关注点从业务逻辑中解耦，提升代码可维护性；通过统一 CORS 配置可解决前后端分离开发中的跨域访问问题，便于联调与部署。</p>
