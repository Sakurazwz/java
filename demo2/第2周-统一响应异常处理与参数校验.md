## 第一部分：为什么需要统一响应（10分钟）

### 问题场景

在没有统一响应的情况下，不同的接口可能返回不同的格式：

```java
// 接口1：返回字符串
@GetMapping("/user1")
public String getUser1() {
    return "张三";
}

// 接口2：返回对象
@GetMapping("/user2")
public User getUser2() {
    return new User("张三", 20);
}

// 接口3：返回Map
@GetMapping("/user3")
public Map<String, Object> getUser3() {
    Map<String, Object> map = new HashMap<>();
    map.put("name", "张三");
    map.put("age", 20);
    return map;
}

// 接口4：发生异常时返回错误
@GetMapping("/error")
public String error() {
    throw new RuntimeException("出错了！");
}
```

**问题**：
- 前端无法统一处理响应
- 错误信息格式不统一
- 无法携带额外信息（如时间戳、请求ID）

### 统一响应的优势

1. **前后端分离友好**：前端可以统一解析响应
2. **易于扩展**：可以添加全局字段（时间戳、追踪ID）
3. **错误处理统一**：成功和失败都返回相同的结构
4. **易于监控和日志**：统一格式便于日志分析

## 第二部分：实现统一响应结构（20分钟）

### 步骤1：创建响应码枚举

创建 `com.example.demo.common.ResultCode.java`：

```java
package com.example.demo.common;

import lombok.Getter;

/**
 * 响应码枚举
 */
@Getter
public enum ResultCode {

    /* 成功 */
    SUCCESS(200, "操作成功"),

    /* 客户端错误 4xx */
    BAD_REQUEST(400, "请求参数错误"),
    UNAUTHORIZED(401, "未授权"),
    FORBIDDEN(403, "禁止访问"),
    NOT_FOUND(404, "资源不存在"),
    METHOD_NOT_ALLOWED(405, "请求方法不支持"),

    /* 服务器错误 5xx */
    INTERNAL_SERVER_ERROR(500, "服务器内部错误"),
    SERVICE_UNAVAILABLE(503, "服务暂不可用"),

    /* 业务错误码 1xxx */
    USER_NOT_FOUND(1001, "用户不存在"),
    USER_ALREADY_EXISTS(1002, "用户已存在"),
    INVALID_PASSWORD(1003, "密码错误"),
    INVALID_EMAIL(1004, "邮箱格式不正确"),
    INVALID_PHONE(1005, "手机号格式不正确");

    /**
     * 响应码
     */
    private final int code;

    /**
     * 响应消息
     */
    private final String message;

    ResultCode(int code, String message) {
        this.code = code;
        this.message = message;
    }
}
```

**说明**：
- 使用枚举统一管理响应码
- 遵循 HTTP 状态码规范（2xx成功、4xx客户端错误、5xx服务器错误）
- 自定义业务错误码使用 1xxx 避免冲突

### 步骤2：创建统一响应类

创建 `com.example.demo.common.Result.java`：

```java
package com.example.demo.common;

import lombok.Data;

import java.io.Serializable;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

/**
 * 统一响应结果
 *
 * @param <T> 数据类型
 */
@Data
public class Result<T> implements Serializable {

    private static final long serialVersionUID = 1L;

    /**
     * 响应码
     */
    private int code;

    /**
     * 响应消息
     */
    private String message;

    /**
     * 响应数据
     */
    private T data;

    /**
     * 时间戳
     */
    private String timestamp;

    /**
     * 请求ID（用于链路追踪）
     */
    private String traceId;

    public Result() {
        this.timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    }

    public Result(int code, String message, T data) {
        this();
        this.code = code;
        this.message = message;
        this.data = data;
    }

    public Result(ResultCode resultCode, T data) {
        this();
        this.code = resultCode.getCode();
        this.message = resultCode.getMessage();
        this.data = data;
    }

    /**
     * 成功返回（无数据）
     */
    public static <T> Result<T> success() {
        return new Result<>(ResultCode.SUCCESS, null);
    }

    /**
     * 成功返回（有数据）
     */
    public static <T> Result<T> success(T data) {
        return new Result<>(ResultCode.SUCCESS, data);
    }

    /**
     * 成功返回（自定义消息）
     */
    public static <T> Result<T> success(String message, T data) {
        Result<T> result = new Result<>(ResultCode.SUCCESS, data);
        result.setMessage(message);
        return result;
    }

    /**
     * 失败返回（使用错误码枚举）
     */
    public static <T> Result<T> error(ResultCode resultCode) {
        return new Result<>(resultCode, null);
    }

    /**
     * 失败返回（自定义错误码和消息）
     */
    public static <T> Result<T> error(int code, String message) {
        return new Result<>(code, message, null);
    }

    /**
     * 失败返回（自定义错误码枚举和数据）
     */
    public static <T> Result<T> error(ResultCode resultCode, T data) {
        return new Result<>(resultCode, data);
    }

    /**
     * 失败返回（仅消息）
     */
    public static <T> Result<T> error(String message) {
        return new Result<>(ResultCode.BAD_REQUEST.getCode(), message, null);
    }

    /**
     * 判断是否成功
     */
    public boolean isSuccess() {
        return this.code == ResultCode.SUCCESS.getCode();
    }
}
```

**说明**：
- 使用泛型 `<T>` 支持任意类型的数据
- 提供多个静态工厂方法，方便调用
- 自动添加时间戳
- 预留 traceId 字位用于链路追踪

## 第三部分：全局异常处理（30分钟）

### 问题场景

在实现统一响应后，我们还需要处理异常情况：

```java
@GetMapping("/exception")
public Result<String> exception() {
    int i = 1 / 0;  // 会抛出 ArithmeticException
    return Result.success();
}
```

**问题**：
- 异常会直接返回给前端，格式不统一
- 前端无法统一处理
- 敏感信息可能泄露（如数据库错误）

### 解决方案：全局异常处理

使用 `@ControllerAdvice` + `@ExceptionHandler` 实现全局异常捕获，统一返回格式。

---

### 步骤1：创建自定义业务异常

创建 `com.example.demo.exception.BusinessException.java`：

```java
package com.example.demo.exception;

import com.example.demo.common.ResultCode;
import lombok.Getter;

/**
 * 业务异常
 */
@Getter
public class BusinessException extends RuntimeException {

    /**
     * 错误码
     */
    private final int code;

    /**
     * 错误消息
     */
    private final String message;

    public BusinessException(String message) {
        super(message);
        this.code = 400;
        this.message = message;
    }

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
        this.message = message;
    }

    public BusinessException(ResultCode resultCode) {
        super(resultCode.getMessage());
        this.code = resultCode.getCode();
        this.message = resultCode.getMessage();
    }

    public BusinessException(ResultCode resultCode, String message) {
        super(message);
        this.code = resultCode.getCode();
        this.message = message;
    }
}
```

**说明**：
- 继承 `RuntimeException`（无需声明抛出）
- 携带错误码和错误消息
- 提供多种构造方法

### 步骤2：创建全局异常处理器

创建 `com.example.demo.exception.GlobalExceptionHandler.java`：

```java
package com.example.demo.exception;

import com.example.demo.common.Result;
import com.example.demo.common.ResultCode;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpStatus;
import org.springframework.validation.BindException;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;

import jakarta.servlet.http.HttpServletRequest;
import java.util.stream.Collectors;

/**
 * 全局异常处理器
 */
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    /**
     * 业务异常
     */
    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusinessException(BusinessException e, HttpServletRequest request) {
        log.warn("业务异常: {}, 请求路径: {}", e.getMessage(), request.getRequestURI());
        return Result.error(e.getCode(), e.getMessage());
    }

    /**
     * 参数校验异常（RequestBody）
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<?> handleValidationException(MethodArgumentNotValidException e) {
        String errorMsg = e.getBindingResult().getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining("; "));
        log.warn("参数校验失败: {}", errorMsg);
        return Result.error(ResultCode.BAD_REQUEST.getCode(), errorMsg);
    }

    /**
     * 参数绑定异常（RequestParam 或 ModelAttribute）
     */
    @ExceptionHandler(BindException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<?> handleBindException(BindException e) {
        String errorMsg = e.getBindingResult().getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining("; "));
        log.warn("参数绑定失败: {}", errorMsg);
        return Result.error(ResultCode.BAD_REQUEST.getCode(), errorMsg);
    }

    /**
     * 非法参数异常
     */
    @ExceptionHandler(IllegalArgumentException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<?> handleIllegalArgumentException(IllegalArgumentException e) {
        log.warn("非法参数: {}", e.getMessage());
        return Result.error(ResultCode.BAD_REQUEST.getCode(), e.getMessage());
    }

    /**
     * 算术异常（如除零错误）
     */
    @ExceptionHandler(ArithmeticException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<?> handleArithmeticException(ArithmeticException e, HttpServletRequest request) {
        log.error("算术异常: {}, 请求路径: {}", e.getMessage(), request.getRequestURI(), e);
        return Result.error(ResultCode.INTERNAL_SERVER_ERROR);
    }

    /**
     * 空指针异常
     */
    @ExceptionHandler(NullPointerException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<?> handleNullPointerException(NullPointerException e, HttpServletRequest request) {
        log.error("空指针异常: {}, 请求路径: {}", e.getMessage(), request.getRequestURI(), e);
        return Result.error(ResultCode.INTERNAL_SERVER_ERROR.getCode(), "系统内部错误");
    }

    /**
     * 其他未捕获的异常
     */
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<?> handleException(Exception e, HttpServletRequest request) {
        log.error("系统异常: {}, 请求路径: {}", e.getMessage(), request.getRequestURI(), e);
        return Result.error(ResultCode.INTERNAL_SERVER_ERROR);
    }
}
```

**说明**：
- `@RestControllerAdvice` = `@ControllerAdvice` + `@ResponseBody`
- `@ExceptionHandler` 指定处理的异常类型
- `@ResponseStatus` 设置 HTTP 状态码
- 使用 `log` 记录异常日志
- 按照异常粒度从细到粗处理（子类异常在前）

## 第四部分：参数校验（30分钟）

### 为什么需要参数校验？

在开发中，我们需要对用户输入进行校验：

**传统方式**（繁琐）：

```java
@PostMapping("/register")
public Result<String> register(String username, String email, String age) {
    if (username == null || username.length() < 3 || username.length() > 20) {
        return Result.error("用户名长度必须在3-20之间");
    }
    if (email == null || !email.matches("^[A-Za-z0-9+_.-]+@(.+)$")) {
        return Result.error("邮箱格式不正确");
    }
    if (age == null || age < 1 || age > 120) {
        return Result.error("年龄必须在1-120之间");
    }
    // 业务逻辑...
    return Result.success();
}
```

**问题**：
- 代码冗长、可读性差
- 校验逻辑和业务逻辑混合
- 不易维护

**使用参数校验**（优雅）：

```java
@PostMapping("/register")
public Result<String> register(@Valid @RequestBody UserRegisterDTO dto) {
    // 业务逻辑...
    return Result.success();
}
```

### 步骤2：了解常用校验注解

| 注解 | 说明 | 示例 |
|------|------|------|
| `@Null` | 必须为 null | `@Null private String value;` |
| `@NotNull` | 必须不为 null | `@NotNull private String name;` |
| `@NotEmpty` | 必须不为空（用于字符串、集合） | `@NotEmpty private String name;` |
| `@NotBlank` | 必须不为空白（去除首尾空格后长度>0） | `@NotBlank private String name;` |
| `@Min(value)` | 必须大于等于指定值 | `@Min(18) private Integer age;` |
| `@Max(value)` | 必须小于等于指定值 | `@Max(120) private Integer age;` |
| `@Size(min, max)` | 长度必须在指定范围内 | `@Size(min=3, max=20) private String name;` |
| `@Email` | 必须是合法邮箱 | `@Email private String email;` |
| `@Pattern(regexp)` | 必须匹配正则表达式 | `@Pattern(regexp="^1[3-9]\\d{9}$") private String phone;` |
| `@Past` | 必须是过去的日期 | `@Past private Date birthDate;` |
| `@Future` | 必须是未来的日期 | `@Future private Date appointmentDate;` |
| `@Positive` | 必须为正数 | `@Positive private Integer amount;` |
| `@Negative` | 必须为负数 | `@Negative private Integer debt;` |

- `@Validated` - 支持分组

### 步骤1：创建校验注解

创建 `com.example.demo.validation.PasswordStrength.java`：

```java
package com.example.demo.validation;

import jakarta.validation.Constraint;
import jakarta.validation.Payload;
import java.lang.annotation.*;

/**
 * 密码强度校验注解
 * 要求：密码必须包含大小写字母和数字，长度8-20位
 */
@Documented
@Constraint(validatedBy = PasswordStrengthValidator.class)
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
public @interface PasswordStrength {

    /**
     * 错误消息
     */
    String message() default "密码必须包含大小写字母和数字，长度8-20位";

    /**
     * 分组（用于分组校验）
     */
    Class<?>[] groups() default {};

    /**
     * 负载（可用于元数据）
     */
    Class<? extends Payload>[] payload() default {};

    /**
     * 最小长度
     */
    int min() default 8;

    /**
     * 最大长度
     */
    int max() default 20;
}
```

**说明**：
- `@Constraint` - 指定校验器类
- `@Target` - 指定注解可以使用的地方（字段、参数）
- `@Retention` - 注解保留策略（RUNTIME 表示运行时可通过反射获取）

---

### 步骤2：实现校验器

创建 `com.example.demo.validation.PasswordStrengthValidator.java`：

```java
package com.example.demo.validation;

import jakarta.validation.ConstraintValidator;
import jakarta.validation.ConstraintValidatorContext;
import java.util.regex.Pattern;

/**
 * 密码强度校验器
 */
public class PasswordStrengthValidator implements ConstraintValidator<PasswordStrength, String> {

    private int min;
    private int max;

    /**
     * 初始化方法（获取注解参数）
     */
    @Override
    public void initialize(PasswordStrength constraintAnnotation) {
        this.min = constraintAnnotation.min();
        this.max = constraintAnnotation.max();
    }

    /**
     * 校验方法
     *
     * @param value   待校验的值
     * @param context 校验上下文
     * @return true-校验通过，false-校验失败
     */
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        // null 值交给 @NotNull 处理
        if (value == null) {
            return true;
        }

        // 长度校验
        if (value.length() < min || value.length() > max) {
            return false;
        }

        // 正则校验：必须包含大小写字母和数字
        String pattern = "^(?=.*[a-z])(?=.*[A-Z])(?=.*\\d).+$";
        return Pattern.matches(pattern, value);
    }
}
```

**说明**：
- 实现 `ConstraintValidator` 接口
- 第一个泛型是注解类型，第二个是校验值的类型
- `initialize()` 方法可以获取注解参数
- `isValid()` 方法实现校验逻辑

##### 步骤3：实现校验器

创建 `com.example.demo.validation.EnumValueValidator.java`：

```java
package com.example.demo.validation;

import jakarta.validation.ConstraintValidator;
import jakarta.validation.ConstraintValidatorContext;

/**
 * 枚举值校验器
 */
public class EnumValueValidator implements ConstraintValidator<EnumValue, String> {

    private Class<? extends Enum<?>> enumClass;

    /**
     * 初始化方法（获取注解参数）
     */
    @Override
    public void initialize(EnumValue constraintAnnotation) {
        this.enumClass = constraintAnnotation.enumClass();
    }

    /**
     * 校验方法
     */
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        // null 值交给 @NotNull 处理
        if (value == null || value.isEmpty()) {
            return true;
        }

        // 遍历枚举常量，检查是否匹配枚举名称
        for (Enum<?> enumConstant : enumClass.getEnumConstants()) {
            if (enumConstant.name().equals(value)) {
                return true;  // 匹配成功
            }
        }

        return false;  // 不匹配
    }
}
```

**工作原理**：
- 校验器会遍历枚举的所有常量
- 将输入的字符串与枚举名称（`MALE`、`FEMALE`、`OTHER`）比较
- 如果匹配则校验通过，否则返回 `false`

##### 常见错误

**错误1：枚举名称不匹配**

```java
public enum Gender {
    MALE, FEMALE, OTHER
}

// 输入: "Male" (注意大小写)
// 结果: 校验失败，因为枚举名是 "MALE"（全大写），不是 "Male"
```

**解决方案**：使用大写枚举名，或者在注解中添加额外的处理逻辑。

---

**错误2：忘记添加枚举类依赖**

```java
@EnumValue(enumClass = Gender.class, ...)  // Gender 类找不到
private String gender;
```

**解决方案**：确保 `Gender` 类在类路径下，并且已正确导入。

---

##### 扩展：匹配枚举的其他属性

如果需要匹配枚举的 `code` 属性（1、0、2），可以修改 `isValid` 方法：

```java
@Override
public boolean isValid(String value, ConstraintValidatorContext context) {
    if (value == null || value.isEmpty()) {
        return true;
    }

    // 先尝试按名称匹配
    for (Enum<?> enumConstant : enumClass.getEnumConstants()) {
        if (enumConstant.name().equals(value)) {
            return true;
        }
    }

    // 再尝试按 code 匹配（如果枚举有 code 属性）
    try {
        Integer valueCode = Integer.parseInt(value);
        for (Enum<?> enumConstant : enumClass.getEnumConstants()) {
            Gender gender = (Gender) enumConstant;
            if (gender.getCode().equals(valueCode)) {
                return true;
            }
        }
    } catch (NumberFormatException e) {
        // 忽略，不是数字
    }

    return false;
}
```

这样就可以接受 `1`、`0`、`2` 作为输入值了。

2. **文档说明**：在注解的 `message` 中说明有效的值
3. **错误提示友好**：明确告知哪些值是有效的

## 第六部分：分组校验（15分钟）

### 场景描述

不同场景下，对同一字段的校验规则可能不同。

**示例**：
- 新增用户：所有字段都需要校验
- 更新用户：只需要校验 ID 和部分字段

---

### 步骤1：定义校验分组

创建 `com.example.demo.validation.ValidationGroup.java`：

```java
package com.example.demo.validation;

import jakarta.validation.groups.Default;

/**
 * 校验分组
 */
public interface ValidationGroup {

    /**
     * 新增分组
     */
    interface Create extends Default {
    }

    /**
     * 更新分组
     */
    interface Update extends Default {
    }

    /**
     * 删除分组
     */
    interface Delete {
    }
}
```

**说明**：
- 继承 `jakarta.validation.groups.Default` 可以包含默认校验
- 分组接口只是一个标记，不需要实现

---

### 步骤2：创建 DTO 并使用分组

创建 `com.example.demo.dto.UserUpdateDTO.java`：

```java
package com.example.demo.dto;

import com.example.demo.validation.ValidationGroup;
import lombok.Data;

import jakarta.validation.constraints.*;

/**
 * 用户更新 DTO
 */
@Data
public class UserUpdateDTO {

    /**
     * 用户ID（更新时必填）
     */
    @NotNull(message = "用户ID不能为空", groups = ValidationGroup.Update.class)
    private Long id;

    /**
     * 用户名（新增时必填，更新时可选）
     */
    @NotBlank(message = "用户名不能为空", groups = ValidationGroup.Create.class)
    @Size(min = 3, max = 20, message = "用户名长度必须在3-20个字符之间", groups = {ValidationGroup.Create.class, ValidationGroup.Update.class})
    private String username;

    /**
     * 邮箱（新增时必填，更新时可选）
     */
    @NotBlank(message = "邮箱不能为空", groups = ValidationGroup.Create.class)
    @Email(message = "邮箱格式不正确", groups = {ValidationGroup.Create.class, ValidationGroup.Update.class})
    private String email;

    /**
     * 年龄（两个场景都校验）
     */
    @Min(value = 1, message = "年龄必须大于0")
    @Max(value = 120, message = "年龄必须小于120")
    private Integer age;
}
```

## 第七部分：快速失败（可选）

### 场景描述

默认情况下，参数校验会返回所有错误。如果希望在第一个错误时就返回，可以配置"快速失败"模式。

---

### 步骤1：配置快速失败

创建 `com.example.demo.config.ValidationConfig.java`：

```java
package com.example.demo.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.validation.beanvalidation.LocalValidatorFactoryBean;
import org.springframework.validation.beanvalidation.MethodValidationPostProcessor;

import jakarta.validation.Validator;

/**
 * 参数校验配置
 */
@Configuration
public class ValidationConfig {

    /**
     * 配置快速失败模式（第一个校验失败就返回）
     */
    @Bean
    public Validator validator() {
        LocalValidatorFactoryBean validator = new LocalValidatorFactoryBean();
        // Hibernate Validator 8.x 使用字符串配置
        validator.getValidationPropertyMap().put("hibernate.validator.fail_fast", "true");
        return validator;
    }

    @Bean
    public MethodValidationPostProcessor methodValidationPostProcessor() {
        MethodValidationPostProcessor processor = new MethodValidationPostProcessor();
        processor.setValidator(validator());
        return processor;
    }
}
```

**说明**：
- `hibernate.validator.fail_fast` 是快速失败的配置属性名
- 通过 `getValidationPropertyMap()` 设置配置属性
- 这种方式兼容 Spring Boot 3.x / Hibernate Validator 8.x

## 常见问题解答

### Q1：`@Valid` 和 `@Validated` 有什么区别？

| 特性 | `@Valid` | `@Validated` |
|------|----------|--------------|
| 来源 | jakarta.validation | org.springframework.validation.annotation |
| 分组校验 | 不支持 | 支持 |
| 位置 | 方法参数、字段 | 方法参数、类 |
| 嵌套校验 | 支持 | 支持 |

**建议**：
- 使用 `@Validated` 在类上启用分组校验
- 使用 `@Valid` 在方法参数上触发嵌套校验

---

### Q2：为什么 `@RequestParam` 校验不生效？

**原因**：需要在 Controller 类上添加 `@Validated` 注解。

```java
@RestController
@RequestMapping("/user")
@Validated  // 必须添加
public class UserController {

    @GetMapping("/query")
    public Result<String> query(
            @NotBlank(message = "用户名不能为空")
            String username) {
        return Result.success(username);
    }
}
```

---

### Q3：如何校验嵌套对象？

**场景**：DTO 中包含另一个对象。

```java
public class UserDTO {
    @NotBlank
    private String username;

    @Valid  // 添加 @Valid 触发嵌套校验
    private Address address;
}

public class Address {
    @NotBlank
    private String province;

    @NotBlank
    private String city;
}
```

**关键**：在嵌套对象字段上添加 `@Valid` 注解。

---

### Q4：如何在校验器中使用 Spring Bean？

**场景**：校验时需要查询数据库（如检查用户名是否已存在）。

**方案**：使用 Spring 的 `LocalValidatorFactoryBean` 并注入 Bean。

```java
@Component
public class UniqueUsernameValidator implements ConstraintValidator<UniqueUsername, String> {

    @Autowired
    private UserRepository userRepository;  // 注入 Service

    @Override
    public boolean isValid(String username, ConstraintValidatorContext context) {
        if (username == null || username.isEmpty()) {
            return true;
        }
        // 查询数据库
        return !userRepository.existsByUsername(username);
    }
}
```

**注意**：确保校验器是 Spring Bean（添加 `@Component`）。

---

### Q5：如何自定义校验错误消息的插值？

**场景**：错误消息需要包含参数值。

```java
@Size(min = 3, max = 20, message = "用户名长度必须在{min}-{max}个字符之间")
private String username;
```

Spring Boot 默认支持消息插值，可以直接使用 `{min}`、`{max}` 等占位符。
